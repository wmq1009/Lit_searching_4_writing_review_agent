##  快速开始

只需四个步骤即可生成您的第一篇综述：
1. **配置模型**：点击右上角齿轮图标，输入您的 API Key。
2. **搜索文献**：在主搜索框输入关键词（如 "Machine learning in lung cancer"）。
3. **选择文献**：从搜索结果中勾选您感兴趣的文章。
4. **生成综述**：点击“Generate Review”，AI 将自动完成总结、起草和格式化。

---

##  配置 LLM 设置

本工具支持多种主流 AI 模型。点击页面右上角的 **Settings (齿轮图标⚙️)** 进行配置：

### 支持的提供商：
*   **Google Gemini (推荐)**: 默认使用 `gemini-3-flash-preview`，速度快且支持原生推理过程显示。
*   **DeepSeek**: 支持 `deepseek-chat`。特别优化了对“思考过程 (Reasoning Content)”的捕获。
*   **OpenAI (GPT)**: 支持 `gpt-4o`、`gpt-4-turbo` 等。
*   **Alibaba Qwen (通义千问)**: 通过阿里云 DashScope 接口调用。
*   **Ollama (本地运行)**: 允许您在本地运行模型（如 `llama3`），保护隐私且完全免费。
    *   *注意：使用 Ollama 时需确保本地已开启 CORS（设置环境变量 `OLLAMA_ORIGINS="*"`）。*

## 文献检索与筛选

### 1. 智能搜索
在搜索框输入您的研究课题。您可以通过下拉菜单筛选 **“近五年”** 或 **“近十年”** 的文献，确保引用的时效性。

### 2. 撤稿文献自动过滤 (Retraction Filter)
为了保证学术严谨性，系统会自动识别并标记 PubMed 中的撤稿文献（Retracted Publication）。
*   在检索阶段，撤稿文献会显示红色警告标识。
*   在生成阶段，系统会**二次强制过滤**，确保撤稿文献绝不会被纳入最终综述。

### 3. 文献预览
点击文章标题可展开查看摘要，帮助您在勾选前快速判断文献相关性。

##  综述生成流程

选择好文献后，点击 **"Generate Review"**，系统将进入自动化工作流：

1.  **文献总结 (Summarizing)**: AI 逐篇分析选定文献的核心观点。
2.  **章节起草 (Drafting)**: 按照标准学术结构（Introduction, Body, Discussion, Conclusion）生成内容。
3.  **引用格式化 (Formatting)**: 根据您选择的样式自动生成参考文献列表。

### 章节预览与编辑
生成的每个章节都会实时显示。您可以：
*   **查看推理过程**：点击 "Reasoning Process" 查看 AI 的思考逻辑。
*   **手动修改**：直接在文本框内编辑生成的内容。
*   **反馈重生成**：如果您对某个章节不满意，可以在下方输入反馈（如“请更多关注临床结果”），然后点击 **"Regenerate Section"**。

##  核心功能详解

### 1. 引用格式支持
支持三种主流学术引用格式：
*   **APA**: 作者-日期制。
*   **Harvard**: 传统的哈佛格式。
*   **Vancouver (温哥华)**: 顺序编码制。

### 2. 格式上标支持 (Superscript)
当选择 Vancouver 格式时，搜索栏下方会出现 **"Superscript"** 切换按钮。
*   **开启后**：正文引用将显示为上标形式，如 `[1]<sup>[1]</sup>`。
*   **关闭后**：显示为平排方括号，如 `[1]`。

### 3. 引用自动转换
系统会自动将原始数据中的占位符（如 `[Paper 1]`）转换为符合所选格式的标准引文（如 `[1]` 或 `(Smith et al., 2023)`）。

## 常见问题 (FAQ)

**Q: 我的 API Key 安全吗？**
A: 您的 API Key 仅保存在浏览器本地状态中，直接与对应的 AI 提供商接口通信，不会上传到我们的服务器。

**Q: 为什么 Ollama 连接失败？**
A: 请检查：
1. Ollama 是否正在运行。
2. 是否设置了 `OLLAMA_ORIGINS="*"`。
3. 尝试使用 `127.0.0.1` 代替 `localhost`。

**Q: 生成的综述可以直接发表吗？**
A: 本工具生成的综述仅供参考和初稿起草。学术论文需要严谨的同行评审和人工校对，请务必核实 AI 生成的所有事实和引用。

**Q: 参考文献列表在哪里？**
A: 在所有章节生成完毕后，系统会自动在最后生成完整的 **"References"** 列表。
